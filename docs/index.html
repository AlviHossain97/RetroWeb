System Architecture: The Hardware Split

PiStation is fundamentally a two-host architecture. It divides workloads across a local network to maximize performance, playing to the strengths of both devices.

📺 Raspberry Pi 3 (The Client)

Specs: Quad-core ARM Cortex-A53 @ 1.2 GHz, 1 GB LPDDR2, VideoCore IV GPU.

Role: Attached directly to the TV. Handles low-latency native retro emulation via RetroArch (lr-mgba, lr-snes9x), H.264/HEVC hardware decoding for Kodi media playback, and runs Moonlight as a thin client to receive the streamed AI dashboard.

💻 Laptop (The Powerhouse Server)

Specs: x86_64, RTX 3070 8 GB VRAM, 16 GB RAM.

Role: Hosts everything GPU-bound or demanding. Runs the FastAPI analytics backend, MySQL database, Jellyfin media server, Parakeet STT (CUDA fp16), Kokoro TTS (ONNX), and the React 19 PWA. Uses Sunshine NVENC to stream the dashboard to the Pi.

The Three Interaction Loops

PiStation unifies three separate experiences into a seamless TV interface, all launched from the EmulationStation carousel.

1. Native Gaming

EmulationStation launches RetroArch natively. Game ROMs do not live on the Pi; they are streamed from the laptop via a Samba/CIFS mount (/mnt/laptop). runcommand hooks (onstart and onend) execute session_logger.py to record gameplay sessions (timestamps, duration, core) to the laptop's MySQL database.

2. Media Playback

Jellyfin acts as a dedicated system entry in EmulationStation. Running Jellyfin.sh spawns a new X server on VT8 and launches Kodi with the Jellyfin-for-Kodi add-on. Video transcoding happens on the laptop's RTX 3070, while the Pi handles H.264/HEVC decode.

3. AI Dashboard

The React 19 PWA runs on the laptop inside a headless Chromium window. Sunshine NVENC encodes the window at 720p30 (8 Mbps) and streams it to the Pi via Moonlight. Controller inputs from the Pi's 8BitDo SN30 Pro are forwarded back to the laptop.

Dashboard Frontend & Emulation Fallback

The central hub is a React 19 + Vite 7 + Tailwind CSS v4 PWA. It provides live analytics, achievement tracking, and voice assistant access.

Key Frontend Features

  • State Management & Storage: Zustand for global app state. Dexie.js (IndexedDB) for browser-local persistent storage.
  • Browser Emulation Route: While production gaming runs on the Pi, the dashboard includes an offline-capable browser emulator powered by Nostalgist (RetroArch WASM).
  • ROM Normalization: Advanced ZIP extraction using @zip.js for dropping ROMs directly into the browser.
  • Cross-Origin Isolation: Strict COOP/COEP headers are enforced via Vite to enable SharedArrayBuffer for multi-threaded WASM libretro cores.

Privacy-First AI Voice Assistant

The PiStation dashboard features a fully local, privacy-first conversational AI. Audio never leaves your local network unless explicitly configured to use external LLMs.

Browser Mic (AudioWorklet) ──> Silence VAD ──> Parakeet STT (:8786) ──> NVIDIA API (LLM) ──> Kokoro TTS (:8787) ──> Browser AudioContext

The Technical Pipeline

  • Voice Capture: An AudioWorklet processes browser audio into PCM16 @ 16 kHz mono. It features built-in Voice Activity Detection (VAD) to auto-submit chunks when the user stops speaking.
  • Local STT & TTS: Speech-to-text is handled by NVIDIA's Parakeet TDT 1.1B (fp16 on CUDA). Text-to-speech uses Kokoro ONNX. Audio returns to the browser as PCM16 @ 22.05 kHz mono.
  • VAD Suppression: The frontend actively suppresses VAD activation while TTS audio is playing to prevent the assistant from transcribing its own voice.
  • Activation Modes: auto_near_field (continuous listening with VAD), headset (tighter thresholds), and push_to_talk (hold 'T').
  • Web Grounding & Vision: Can search the web via Tavily/SearXNG and process images (emulator screenshots) using multi-modal LLMs (like LLaVA or Qwen).

Original IP-Safe Homebrew Corpus

To ensure a completely copyright-clean submission, three original Game Boy Advance homebrew games were developed from scratch. They follow a strict Python-to-GBA-native pipeline: rapid gameplay iteration in Python/Pygame, followed by a from-scratch C/C++ rewrite targeting GBA constraints.

🏎️ Red Racer

Language: C

A high-speed traffic dodging game. Written in C using devkitARM. Features fixed-point 16.16 math, object pooling, and strict OAM sprite management. Sourced assets from TMD Studios (itch.io).

⚔️ Mythical

Language: C++

A top-down adventure exploration game. 100% project-author original assets. Used a C++ desktop simulator intermediate stage before final GBA compilation.

🏰 Bastion Tower Defence

Language: C++ (Butano Engine)

A grid-based tower defence game. Utilizes the modern Butano C++23 GBA engine. Python prototype port required building a hardware-abstraction layer. Sourced assets from pixel.iwao (itch.io).

Backend Services & Data Layer

FastAPI Backend

Orchestrated by start.mjs, the backend comprises 12 route modules covering sessions, telemetry, AI orchestration, image generation, and WebSocket-based voice chunk streaming.

MySQL 8 Database

Hosted locally via XAMPP. The schema manages sessions, games, devices, achievements, controller_profiles, and ai_conversations.
