Architecture

The artefact is a LAN-cooperating Pi-and-laptop system.

PiStation separates the parts of the experience by hardware suitability. The Pi stays at the TV and handles low-latency console-style interaction. The laptop handles anything GPU-bound, storage-heavy, or AI-heavy, then streams the finished interface to the Pi.

Raspberry Pi 3 client

Hardware: quad-core ARM Cortex-A53 at 1.2 GHz, 1 GB LPDDR2, VideoCore IV GPU, HDMI to TV, 8BitDo SN30 Pro controller.

Role: EmulationStation launcher, RetroArch native emulation, Kodi/Jellyfin playback, and Moonlight thin client.

Interaction model

Three independent loops, one TV-facing experience.

The project is not only a web app. It is a physical system with three routes through the same user-facing launcher: native games, media playback, and the streamed dashboard.

1

Native gaming loop

EmulationStation launches RetroArch directly on the Pi. The original GBA games run through lr-mgba. ROM storage is centralised on the laptop using Samba/CIFS, while gameplay stays native on the Pi for low input latency.

2

Session capture loop

RetroPie's runcommand-onstart.sh and runcommand-onend.sh hooks call session_logger.py, which opens and closes session rows in MySQL through FastAPI. The dashboard's playtime, recently played, and system stats come from this path.

3

Dashboard and media loop

Kodi connects to laptop-hosted Jellyfin for media. The React dashboard runs fullscreen on the laptop and is streamed to the Pi through Sunshine and Moonlight, so the Pi only decodes video instead of rendering a modern PWA.

Dashboard

A React 19 control surface styled like a retro cabinet HUD.

This page borrows the dashboard's dark surface stack, scanlines, red/cyan glow, stat cards, and progress meters. The real dashboard is a Vite 7 + Tailwind v4 PWA with gamepad-friendly navigation and kiosk-ready display.

Analytics core

Total playtime, sessions, recently played games, top games, system share, and live "now playing" state are served from FastAPI and MySQL.

Gamepad tooling

Controller diagnostics and keyboard rebinding are included so the TV setup can be tested from the sofa without falling back to desktop-only controls.

Achievement layer

Seeded achievements track progress and give the dashboard a persistent sense of progression beyond raw session history.

Runtime surface

Frontend: React 19, TypeScript, Vite 7, Tailwind CSS v4, shadcn/ui, Zustand.

Persistence: MySQL for Pi session data, Dexie/IndexedDB for browser-local library and emulator support data.

Streaming: Sunshine captures the laptop window and sends H.264/HEVC to Moonlight on the Pi.

Voice and AI

Local audio, streamed responses, and model routing.

The assistant is designed around privacy and low-latency interaction. Browser audio is chunked locally, speech services run on the laptop, and LLM completions are routed through NVIDIA's API with optional grounding. 

Browser mic AudioWorklet
  -> PCM16 16 kHz mono chunks
  -> FastAPI voice WebSocket
  -> Parakeet TDT 1.1B STT on CUDA (:8786)
  -> NVIDIA LLM completion stream
  -> Kokoro ONNX TTS (:8787)
  -> PCM16 22.05 kHz playback in browser

Voice capture

The browser worklet downsamples microphone input, sends 80 ms frames, and suppresses voice activity detection while TTS is playing so the assistant does not hear itself.

  • AudioWorklet
  • PCM16
  • VAD
  • WebSocket

Backend and data

FastAPI ties Pi telemetry, dashboard data, and AI services together.

The laptop backend is the system's ingestion and orchestration layer. It accepts Pi session posts, serves dashboard views, handles AI context assembly, and proxies voice service calls.

Layer Technology Responsibility
API FastAPI, Python 3.12, Uvicorn Sessions, games, systems, devices, achievements, health, AI, image generation, and voice WebSocket routes.
Database MySQL 8, PyMySQL, DBUtils pool Stores session rows, game metadata, devices, achievements, controller profiles, and AI conversation records.
Orchestrator start.mjs Starts XAMPP, FastAPI, Kokoro, Parakeet, Vite, and Sunshine in the order the laptop stack expects.
Media Jellyfin server + Kodi add-on Laptop hosts the media library and transcodes when needed; the Pi only decodes playback video.
Streaming Sunshine NVENC + Moonlight Converts the laptop dashboard into a low-latency TV interface without forcing the Pi to render React.

Original homebrew corpus

Three IP-safe GBA games built for the artefact.

The project avoids commercial ROM distribution by shipping original Game Boy Advance homebrew. Each game started as a Python prototype, then became a native C or C++ GBA build that runs on the Pi through lr-mgba.

Red Racer

GBA language: C.

A traffic-dodging racer based on rapid Python prototyping, then rewritten for devkitARM with fixed-point logic and sprite/OAM constraints.

  • Python prototype
  • C port
  • GBA ROM

Mythical

GBA language: C++.

A top-down adventure with project-original assets. It demonstrates the full prototype-to-GBA rewrite without third-party visual assets.

  • Original assets
  • C++ port
  • GBA ROM

Bastion Tower Defence

GBA language: C++ with Butano.

A tower defence game that moved from Python + SDL2 into a Butano C++23 GBA build with static pools and hardware-aware rendering.

  • Butano
  • CMake simulator
  • GBA ROM
The Python prototypes are design tools, not transpilation sources. The GBA versions are separate native implementations shaped around the ARM7TDMI CPU, 240x160 display, palette/tile memory, and no-allocation hot paths.

Setup and repository map

One repo, two physical deployment targets.

The laptop component has the heavier setup because it runs the dashboard, backend, AI services, database, media server, and streamer. The Pi component is deployment-focused: RetroPie, Kodi, Moonlight, controller pairing, and hook scripts. 

git clone https://github.com/AlviHossain97/RetroWeb.git PiStation
cd PiStation
npm install
python3 -m pip install -r backend/requirements.txt
cp backend/.env.example backend/.env
npm start
