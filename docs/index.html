PiStation

A two-host retro gaming and media platform powered by Raspberry Pi 3 and a local AI-driven laptop backend.

BSc Computer Science final-year artefact, University of Roehampton 2025–26.

System Architecture

PiStation is built around a two-host split architecture, dividing workloads across a local network to maximize performance where it counts. The Pi 3 is strong at low-latency native emulation but weak at GPU-bound work (modern web stacks, AI inference, video transcoding). The laptop is strong at GPU work but weak at being attached to a TV.

📺 Raspberry Pi 3 (at the TV)

Hardware: Quad-core ARM, 1 GB RAM, no discrete GPU.

Role: TV-attached client. Handles native retro emulation (RetroArch / lr-mgba), Kodi media playback, and runs the Moonlight thin-client to receive the streamed dashboard.

💻 Laptop (The Backend)

Hardware: x86_64, RTX 3070 8 GB VRAM, 16 GB RAM.

Role: Powerhouse server. Runs the Analytics FastAPI backend, MySQL database, Jellyfin media server, Parakeet STT, Kokoro TTS, local LLMs, and the React 19 PWA dashboard streamed via Sunshine NVENC.

The Three Interaction Loops

PiStation unifies three separate interaction loops into a seamless TV experience:

1. Native Gaming

EmulationStation launches RetroArch natively on the Pi. Game ROMs are read from the laptop via a CIFS mount. runcommand hooks fire before and after play to log session metadata (timestamps, core, duration) to the laptop's FastAPI backend.

2. Media Streaming

Jellyfin acts as a system entry in EmulationStation. Selecting it launches Kodi with the Jellyfin add-on. Video transcoding runs on the laptop's RTX 3070, while the Pi handles H.264/HEVC playback.

3. AI Dashboard

The React 19 PWA runs locally on the laptop inside a headless Chromium window. Sunshine NVENC captures the window and streams it to the Pi via Moonlight. The Pi forwards controller/keyboard inputs back to the laptop.

Frontend & Emulation Engine

The React dashboard serves as a fully client-side PWA. Emulation within the browser is powered by Nostalgist, wrapping RetroArch compiled to WebAssembly.

Key Features

  • State Management: Zustand for global state, Dexie (IndexedDB) for all persistent storage (SRAM saves, BIOS files, user preferences).
  • ROM Normalization: Advanced ZIP extraction using @zip.js and automatic system detection based on file extensions.
  • Saves & BIOS: Auto-saving every 60 seconds. Drag-and-drop BIOS file validation with MD5 hashing.
  • Netplay: Serverless WebRTC P2P multiplayer via SDP offer/answer exchanges over data channels.
  • Cross-Origin Isolation: Relies on SharedArrayBuffer for multi-threaded WASM cores, requiring strict COOP/COEP headers.

Privacy-First AI Voice Assistant

A fully local, privacy-first conversational AI feature integrated into the PWA. No data leaves the user's machine.

The Voice Pipeline

Microphone (Browser) -> Silence Detection -> Whisper STT (:8786) -> Ollama LLM (:11434) -> Kokoro TTS (:8787) -> Audio Playback
  • Silence Detection: An AudioContext AnalyserNode monitors frequency to detect when the user stops speaking, automatically sending the audio chunk.
  • Clause-Level Streaming TTS: As the LLM generates tokens, text is buffered and flushed to the Kokoro TTS engine at sentence and clause boundaries to minimize perceived latency.
  • Vision Capabilities: Hitting the "Ask AI" button takes a base64 PNG screenshot of the emulator canvas and sends it to a multimodal model (like LLaVA).
  • Walkthrough Mode: The assistant can automatically capture and analyze a screenshot every 30 seconds to provide real-time game coaching.
  • Personas: You can select personalities like "Speedrunner", "Historian", or "90s Store Clerk" to alter the system prompt.

Original Homebrew Games

To ensure a clean, copyright-free test corpus for the university submission, three original Game Boy Advance homebrew games were developed alongside the platform. They follow a Python prototyping to GBA-native C/C++ build pipeline.

🏎️ Red Racer

Written in C. Ported from an initial Python prototype.

⚔️ Mythical

Written in C++. Ported from a Python prototype. Uses all project-author original assets.

🏰 Bastion Tower Defence

Written in Butano (C++). Prototyped using Python + SDL2.

The Python-to-GBA pipeline required solving for fixed-point math, 240x160 framebuffer constraints, memory management without dynamic allocation, and custom ROM mapping.
